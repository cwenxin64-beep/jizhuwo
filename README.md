# 请记住我 Recuérdame 小程序原型

## 项目简介

这是一个宠物纪念与情绪陪伴类微信小程序原型。当前首页只保留中间主屋插画，顶部信息、数据条、底部导航和其他功能页都由原生小程序代码绘制。

## 技术架构

- 使用原生微信小程序结构。
- `app.json` 管理主页面、功能分包、窗口和底部导航。
- `app.js` 统一计算真机顶部胶囊和底部安全区距离，页面按这个距离避让。
- `app.wxss` 提供统一颜色、卡片、按钮和页面样式。
- `custom-tab-bar` 仍保留注册能力，但截图页默认隐藏它，避免出现两个底部导航。
- `pages` 下每个功能页使用 WXML/WXSS 绘制页面结构、卡片、按钮和插画。
- `images/*.png` 只保留旧参考资源，打包时整体排除；首页使用压缩后的 `assets/home/home-room-scene.jpg`。
- `assets/game/` 预留小游戏像素素材路径，当前先用 Canvas 色块绘制。
- `pages/livingRoom/assets/` 保存客厅分包实际使用的背景、对话卡片、按钮图标和轻量粒子。
- `pages/bath/assets/` 保存洗澡分包实际使用的浴室场景和透明背景黑橙小狗。
- `pages/cook/assets/` 保存吃饭分包实际使用的背景、小狗和食物图片。
- `pages/walk/assets/` 保存出门准备页的花桥门口、背包和零食原始照片压缩版。
- `assets/album/` 和 `assets/market/` 保存从 Figma 对应图片框下载并压缩的档案、商品照片。
- `assets/navigation/` 保存 Figma 底部导航的四组普通态与选中态图标，`assets/common/` 保存共用头像。
- `cloudfunctions/generatePetImage/` 负责调用 Seedream，并将生成结果转存到 CloudBase。

## 本地运行方法

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择当前文件夹：`outputs/huahua-mini-program`。
4. AppID 可使用测试号或微信开发者工具提供的测试能力。

## 部署方法

1. 在微信开发者工具中绑定正确的 AppID 和 CloudBase 环境。
2. 右键 `cloudfunctions/generatePetImage`，选择“上传并部署：云端安装依赖”。
3. 在 CloudBase 中将函数超时设为 180 秒、内存设为 512MB，并配置环境变量 `ARK_API_KEY`。
4. 小程序代码和云函数部署完成后，再上传并提交微信审核。

详细配置见 `cloudfunctions/generatePetImage/README.md`。API Key 不得写入小程序或代码文件。

## 测试方法

- 在微信开发者工具中打开项目。
- 检查打开项目后是否直接进入「首页｜花桥小屋」真实贴图页。
- 检查 4 个底部主入口是否能通过图片内导航切换。
- 检查底部导航是否使用 Figma 图标、贴底显示，并正确切换四个选中态。
- 检查首页右下角书本按钮、门口花圈是否都能进入出门准备页。
- 检查首页厨房、厕所、客厅区域是否能分别进入做饭、洗澡、客厅互动页。
- 检查客厅互动页中小狗、相框、沙发、玩具球、窗户是否都有轻量反馈，并确认没有叠出第二只狗。
- 检查出门准备页点击“灵感漂流”是否进入像素跑酷小游戏。
- 检查跑酷游戏是否先显示规则说明，点击“开始奔跑”后才开始移动。
- 检查出门准备页的“准备出发”是否能进入跑步页。
- 检查出门准备页背包、零食和心意按钮是否有状态反馈。
- 检查相册页筛选按钮、回忆卡片和“形象实验室”入口是否正常。
- 检查形象实验室能否上传照片、切换三种风格、生成图片并点击查看大图。
- 检查档案页年度大图、5 张回忆图，以及集市页横幅和4张商品图是否正常显示。
- 检查洗澡游戏是否能按“安抚、打湿、起泡、冲净、擦干、吹毛、奖励”的顺序完成。
- 检查洗澡游戏是否支持点选工具、拖拽工具、安静模式、重新洗澡和返回小屋。
- 检查洗澡护理小卡保存后，档案页是否出现“洗得香香的一天”。
- 检查吃饭小游戏是否能跳跃、收集安全食物、避开危险食物和木箱，并能正常重新开始和返回小屋。
- 检查跑步、实验室、集市、我的页面按钮是否都有页面内反馈。
- 检查真机预览时顶部标题、返回键和底部按钮不被系统胶囊或底部手势条遮挡。
- 检查真机上传时主包低于 2MB，所有功能分包均可正常加载。
