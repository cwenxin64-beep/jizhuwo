# 架构说明

## 文件职责

- `project.config.json`：微信开发者工具项目配置。
- `app.js`：小程序全局入口，保存应用名，并计算真机顶部胶囊和底部安全区距离。
- `app.json`：注册主页面、功能分包、窗口样式和自定义底部导航。
- `app.wxss`：全局视觉规则，包括页面背景、卡片、按钮、标签和通用布局。
- `sitemap.json`：小程序索引规则。
- `images/*.png`：保留旧参考图，打包时整体排除。
- `assets/home/home-room-scene.jpg`：首页主屋场景的高质量压缩图。
- `assets/game/*`：预留小游戏像素素材路径，当前 Canvas 先使用色块绘制。
- `pages/livingRoom/assets/*`：客厅分包使用的场景背景、对话卡片、按钮图标和轻量粒子。
- `pages/bath/assets/*`：洗澡分包使用的浴室场景图和透明背景黑橙小狗。
- `pages/cook/assets/*`：吃饭分包使用的背景、小狗和食物素材。
- `pages/walk/assets/*`：出门准备分包使用的门口主图、背包和零食照片。
- `assets/album/*`：从 Figma 档案画板下载的年度大图与5张回忆照片。
- `assets/market/*`：从 Figma 集市画板下载的横幅与4张商品照片。
- `assets/navigation/*`：Figma 底部导航图标的普通态和选中态。
- `assets/common/*`：主页面共用的 Figma 小狗头像。
- `custom-tab-bar/index.*`：保留小程序 tabBar 注册能力，当前截图页默认隐藏。
- `pages/home/home.*`：中间插画 + 原生顶部信息和底部导航版「首页｜花桥小屋」，也是当前默认打开页。
- `pages/walk/walk.*`：原生代码复刻版出门准备页，使用 Figma 原照片组成主图和行囊卡片。
- `pages/run/run.*`：原生代码复刻版跑步页，包含 Figma 小狗主图、脚印路径、音乐、装备和收藏反馈。
- `pages/run/assets/*`：跑步分包使用的奔跑小狗照片、三枚脚印和主按钮图标。
- `pages/runGame/runGame.*`：原生 Canvas 像素跑酷小游戏页，开局先暂停并显示与实际判定一致的规则卡。
- `pages/cook/cook.*`：原生 Canvas 吃饭小游戏，包含跳跃、食物与障碍物生成、碰撞判定、计分和结算。
- `pages/bath/bath.*`：原生洗澡小游戏，包含七步护理、工具点选/拖拽、状态进度、安静模式、护理结算和档案保存。
- `pages/living/living.*`：原生代码复刻版客厅互动页。
- `pages/livingRoom/livingRoom.*`：素材版陪伴式客厅互动页，保留背景图里的固定布丁，只叠加透明热区、光晕、爱心、花瓣和睡眠提示。
- `pages/album/album.*`：宠物相册页，包含筛选和回忆详情展开。
- `pages/lab/lab.*`：形象实验室页，负责上传照片、选择风格、调用云函数和显示生成结果。
- `cloudfunctions/generatePetImage/*`：从 CloudBase 读取输入照片，调用 Seedream，并将生成图转存回云存储。
- `pages/market/market.*`：集市页，包含分类切换、加入心愿单和心愿单面板。
- `pages/profile/profile.*`：我的页，包含档案、开关、隐私、数据和订单说明面板。

## 调用关系

- `app.json` 将首页、档案、集市、我的注册在主包，其余 8 个功能页注册为独立分包。
- 4 个主页面仍通过 `wx.switchTab` 切换：首页、档案、集市、我的。
- 首页隐藏真实自定义 tabBar，使用首页内部代码绘制底部导航；档案、集市、我的恢复使用代码自定义 tabBar。
- 首页通过右下角书本入口和门口花圈入口 `wx.navigateTo` 进入出门准备页。
- 首页通过厨房、厕所、客厅透明点击区分别进入做饭、洗澡、陪伴式客厅互动页。
- 洗澡小游戏完成后使用本地存储保存护理记录，档案页在 `onShow` 时读取并展示最新记录。
- 出门准备页通过“牵好啦，出发”按钮进入跑步页。
- 出门准备页通过“灵感漂流”进入像素跑酷小游戏页。
- 跑酷页在 Canvas 初始化后先绘制静态场景，用户确认规则并点击开始按钮后才启动动画循环。
- 出门准备页的步数兑换由页面数据管理步数、金币和动画锁，兑换时滚动数字并播放箭头流光与金色粒子。
- 跑步页通过中间主按钮回到首页。
- 相册页通过右上角透明点击区进入形象实验室。
- 形象实验室通过 `wx.navigateBack` 返回上一页。
- 形象实验室先上传照片，再调用 `generatePetImage`；云函数从环境变量读取密钥并返回 CloudBase 图片地址。
- 吃饭小游戏通过首页厨房热区进入，完成或失败后可重新开始，也可返回首页。
- 洗澡、实验室、集市、我的、跑步和档案页的按钮都改为页面内反馈，原因是空提示会让用户误以为功能没做。
- 洗澡 HTML 不通过 `web-view` 接入，而是迁移成原生小程序，原因是原生触摸事件、安全区适配和返回流程更稳定。
- 各页面从 `app.js` 读取安全区距离，避免顶部返回键、标题和底部按钮被系统区域遮挡。

## 关键设计决定

- 采用原生微信小程序，而不是 Web 或跨端框架，原因是用户明确要求先按小程序来。
- 保留自定义底部导航但默认隐藏，原因是当前页面已经把 Figma 导航裁进整页贴图，继续显示真实 tabBar 会出现重复导航。
- 将 `pages/home/home` 放在页面列表第一位，原因是用户要求截图里的「首页｜花桥小屋」成为默认首页。
- 首页只保留中间主屋插画为图片，顶部信息、数据条和底部导航改为代码绘制，原因是整页截图在真机会被拉伸或裁掉导航。
- 像素跑酷小游戏使用原生 Canvas 2D，不引入游戏引擎，原因是当前只是课程原型小游戏入口，轻量实现更容易维护。
- 吃饭小游戏同样迁移为原生 Canvas 2D，不使用 `web-view`，原因是触摸、返回和安全区适配更稳定。
- 客厅互动页不做强游戏循环，只做点击热区和轻动画，原因是它承担情绪陪伴核心，节奏需要安静克制。
- 客厅页不再额外叠加狗狗素材，原因是背景图里已经有布丁，固定主角配合轻量反馈更稳，也能避免双狗和抠图边缘。
- 额外补充 `pages/cook/cook` 和 `pages/bath/bath`，原因是这两个页面出现在同一套 Figma 暗色方案里，但不属于底部主导航。
- 视觉只借用暖色、剪纸、烛光、星光桥这些氛围元素，原因是要参考《寻梦环游记》的情绪风格，但避免直接使用受版权保护的角色和标识。
- 形象实验室不放进底部导航，原因是它更像功能页，不是一级主入口。
- 所有页面统一使用真机安全区距离，原因是自定义导航不会自动避让微信胶囊和底部手势条。
- 主页面与功能页拆分打包，原因是旧素材和大场景图会让主包超过微信 2MB 限制；每个功能分包只携带本页实际使用的素材。
- Seedream 密钥只放 CloudBase 云函数环境变量，原因是任何写进小程序包的密钥都能被用户读取。
- AI 生成图立即转存 CloudBase，原因是 Seedream 返回的临时链接会在 24 小时后失效。
